## The Golden Rules for HN

1. **Never be defensive.** Acknowledge valid criticism. Thank people who find bugs.
2. **Be specific, not corporate.** "Great point!" is bad. "You're right — the schema currently doesn't support rotation. It's on the roadmap for v0.3 because [reason]" is good.
3. **Acknowledge competing tools without dismissing them.** The HN crowd will call you out for false comparisons.
4. **If someone says "this already exists," name the difference.** Don't just assert it.
5. **Share data when challenged.** "In my testing" lands better than "our product is better."
6. **Reply within 30 minutes in the first 4 hours.** After that, response time matters less.

## Predicted Comments & Ready Responses

### "Why not just use Excalidraw?"

**Response:**
```
Excalidraw is great — I use it myself for human-driven diagrams. 

The difference: Excalidraw is built for human input (drag, click, type). 
Skissify's input is JSON, which means it can be generated by code or an LLM.

Try getting Claude to draw a floor plan in Excalidraw. 
It can describe one. It can't draw one.

That's the gap Skissify fills: programmatic generation + hand-drawn aesthetic.
```

### "Couldn't you just prompt DALL-E to draw this?"

**Response:**
```
Yes, and I did — extensively during development.

The problems with image generators for technical diagrams:
1. Non-deterministic: prompt "2-bedroom apartment" twice, get two different layouts
2. Text rendering: terrible. DALL-E can't reliably label a room "Bedroom 2"
3. No structured data output: you can't programmatically extract the coordinates, 
   pass them elsewhere, or update one element
4. Hallucination: dimensions are plausible-looking but wrong

Skissify renders from exact coordinates. If you say wall at (50,50) width 200, 
you get that wall, every time, with the correct label. 
The AI generates the JSON; the rendering is deterministic.
```

### "The JSON schema looks verbose. Couldn't this be simpler?"

**Response:**
```
You're right that it's verbose — that's intentional.

Early versions used a more hierarchical, compact schema. LLM success rate on first attempt: ~60%.

After flattening to explicit absolute coordinates with all properties spelled out: ~90%.

LLMs generate better JSON when they don't have to reason about inheritance, 
relative positions, or defaults. Verbose is better for machine-authored JSON.

I wrote about this decision in detail: [link to json-schema blog post]

That said, I'm working on a shorthand for the common cases (a 3-room floor plan 
shouldn't need 100 lines of JSON). Open to ideas if you have them.
```

### "This is just SVG with extra steps"

**Response:**
```
Not wrong — the rendering pipeline does produce SVG-like output.

The difference is the authoring layer. Writing SVG for a floor plan by hand (or having an LLM write it) 
requires knowledge of SVG's coordinate model, transforms, path syntax. 
LLM SVG generation is notoriously unreliable.

The Skissify JSON schema is designed so LLMs can generate it correctly ~90% of the time 
on the first attempt. That's the product: an abstraction layer that LLMs understand.

SVG export is on the roadmap — happy to hear what you'd use it for.
```

### "Why Canvas 2D instead of WebGL/SVG?"

**Response:**
```
Primarily: Canvas 2D is easy to reason about, fast to develop, and more than fast enough 
for this use case. The wobble calculations happen at render time on a 2D context — 
no shader pipeline needed.

SVG would have been the "right" choice for scalability and accessibility, 
but the wobble algorithm (multi-harmonic noise on every path segment) is 
computationally cheaper to express in Canvas 2D than in SVG filters.

Revisiting SVG for the export feature in v0.3. If you have thoughts on the tradeoffs, 
I'm genuinely interested — this is one of the decisions I'm less certain about.
```

## Comment Prioritization

When threads get busy, respond in this order:

1. **Bug reports** — fix fast, thank publicly
2. **Legitimate technical criticism** — respond thoroughly, shows you're serious
3. **"Why not X?"** questions — your clearest positioning moment
4. **Feature requests** — log and acknowledge
5. **General enthusiasm** — brief thanks, ask a question back
6. **"This will never work"** — one clear response, then don't engage further
