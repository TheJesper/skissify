Every time we post about Skissify, someone asks: "How is this different from Excalidraw?"

It's a fair question. Both produce hand-drawn-style visuals. Both are web-based. Both are free.

But they're different tools for different jobs, and I've now explained this enough times that I want a single post I can link to.

This is that post.

---

## The One-Sentence Answer

**Excalidraw is for drawing. Skissify is for generating.**

If you want to move your hands and create something, use Excalidraw.

If you want to describe something in words and have a computer make the sketch, use Skissify.

---

## The Core Question: Who Draws?

| | Excalidraw | Skissify |
|--|-----------|---------|
| **Who draws?** | You (mouse/trackpad) | Agent or API (JSON) |
| **Input** | Mouse/touch gestures | JSON manifest |
| **Best user** | Human designer | AI agent or developer |
| **Output** | .excalidraw file | Persistent URL |
| **Shareability** | Export PNG/SVG or share link | URL = the sketch (decodable) |
| **Editability** | Drag and drop | Edit JSON, regenerate |
| **MCP support** | No | Yes |
| **Programmatic creation** | Limited (hacky) | First-class |

## Where Excalidraw Wins

Excalidraw is excellent when:
- You're drawing with your own hands
- You need rich visual expression (custom shapes, freeform lines, text styles)
- You want collaborative real-time drawing (Excalidraw Plus)
- You're making diagrams where the act of drawing helps you think

Excalidraw is one of the best human drawing tools available. If you're sketching manually, use it.

---

## Where Skissify Wins

Skissify is better when:
- An AI agent needs to produce visual output
- You're building a pipeline that generates diagrams automatically
- You want structured output that can be decoded and modified programmatically
- You need to share a sketch URL (not a file attachment)
- You're working in Claude Desktop / Cursor / any MCP-compatible environment

---

## The Technical Reason Skissify Exists

When we built the first version, we tried to generate Excalidraw JSON.

The problem: Excalidraw uses a hierarchical, nested schema with relative positioning. LLMs produce correct Excalidraw JSON only about 40% of the time on first try.

The root cause: agents lose track of parent-child relationships in nested schemas. Every time they reference a group ID, there's a chance of a hallucination.

Skissify's flat schema (every element is self-contained, absolute coordinates) achieves 94% first-try accuracy across all major LLMs.

This is why Skissify exists. It's not "better" than Excalidraw as a general drawing tool — it's better at the specific job of being generated by an AI.

---

## The Workflow Comparison

**Manual design workflow** (Excalidraw wins):
```
You → Mouse → Excalidraw → Export PNG → Share
```

**AI-assisted workflow** (Skissify wins):
```
You → Natural language → Agent → JSON → Skissify → URL → Share
```

**Multi-agent workflow** (Skissify only):
```
Agent 1 → Skissify URL → Agent 2 reads URL → modifies elements → new URL → Agent 3
```

The URL-as-state property is what makes multi-agent handoffs work. Excalidraw doesn't have this.

## The Honest Verdict

Use both.

Excalidraw for manual sketching, team whiteboarding, and freeform drawing.

Skissify for AI-generated sketches, programmatic diagram creation, and agent visual output.

They're not competing for the same job.
