## Comment Response Protocol

The fastest responders win HN. The rule: **respond within 5 minutes to every comment.**

**Common comment types and how to respond:**

**"How is this different from Excalidraw?"**
> Excalidraw is a collaborative whiteboard for humans. You draw; it assists. Skissify is an API for agents — structured JSON in, sketch out. An AI agent can't use Excalidraw. It can call Skissify with a 3-line fetch. Different jobs.

**"Why not just use Mermaid?"**
> Mermaid makes diagrams. Skissify makes hand-drawn sketches. The wobble algorithm, the pencil texture, the imperfection — that's the product. When you want to communicate "rough idea," a Mermaid diagram looks too polished. A sketch says "this is thinking on paper."

**"The schema seems complex"**
> It's 15 fields. Here's the minimal version that works: [paste 8-line example]. The JSON is meant to be generated by Claude, not hand-written.

**"Could this work with [other model]?"**
> Yes. See the benchmark table: Claude Sonnet 4 (94%), GPT-4o (89%), Haiku (81%), GPT-4o Mini (72%). The schema works with any model that can output valid JSON. Claude is best because it has the best spatial reasoning.

**"What's the business model?"**
> Free during beta. Paid tier for high-volume API usage planned for Week 4. Individual developers always get a free tier.

**"Open source?"**
> The MCP server is MIT licensed. The renderer is currently closed source (the wobble algorithm is where the IP is). Considering open-sourcing the renderer after the beta — what would make that useful for you?
