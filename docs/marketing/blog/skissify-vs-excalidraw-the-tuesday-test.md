# Skissify vs Excalidraw: The Tuesday Test

*This is the version that actually answers the question, not the diplomatic SEO version.*

---

## The Honest Two-Sentence Answer

**Use Excalidraw when:** you're in a room with other humans and everyone is drawing together on the same whiteboard.

**Use Skissify when:** an AI agent is generating the layout, or you need a shareable URL that an AI agent can also read and modify later.

That's the whole comparison. Everything below is elaboration.

---

## What Excalidraw Actually Does

Excalidraw is a collaborative whiteboard. You move your mouse. Shapes appear. Other people move their mice. You see it happen in real-time.

The interaction model is: **human + canvas**.

Excalidraw is excellent for:
- Remote team whiteboarding sessions
- Quick free-form diagrams when you're thinking on the fly
- Any situation where the drawing process *is* the collaboration

---

## What Skissify Actually Does

Skissify is a rendering API. You send JSON. A hand-drawn sketch comes back.

The interaction model is: **data + renderer**.

Skissify is excellent for:
- AI agents that need to produce spatial output
- Automating sketch generation from structured descriptions
- Cases where you want deterministic output (same JSON = same sketch, always)
- Building pipelines where the sketch URL is passed between agents

---

## Why There's No Official Excalidraw MCP Server

This is the key technical distinction. Excalidraw's architecture is canvas-first — it's designed for mouse events, real-time collaboration, and human interaction. Adapting it for AI agent use requires wrapping it in ways that fight the original design.

Community MCP wrappers for Excalidraw exist. They work, but they weren't purpose-built for LLM generation. The schemas are adapted from Excalidraw's internal format, which was designed for human-readable data, not LLM-optimized flat structures.

Skissify's schema was designed specifically to be generated by LLMs:
- Flat array (no nesting depth for LLMs to track)
- Absolute coordinates (no relative positioning)
- Minimal required fields

Result: 94% first-try success with Claude Sonnet 4, vs estimated 60-70% for community Excalidraw wrappers.

## The Workflow That Uses Both

These tools aren't competitors for the same job. The natural workflow uses both:

**Phase 1 — AI Exploration (Skissify)**
Ask Claude to design a space. Claude generates JSON. Skissify renders a rough sketch. You see whether the concept is right. You can do this five times in two minutes.

**Phase 2 — Human Refinement (Excalidraw)**
Once the concept is approximately right, open the sketch in Excalidraw for collaborative human refinement. Add notes. Move shapes. Make precise adjustments with people.

**Phase 3 — Production (Figma, CAD, whatever)**
Once the design is validated, move to the appropriate production tool.

Skissify is the napkin phase. Excalidraw is the whiteboard phase. Figma is the spec phase.

## Feature Comparison

| Feature | Skissify | Excalidraw |
|---------|----------|------------|
| AI/LLM input | ✅ Native (JSON schema optimized for LLMs) | ⚠️ Community wrappers only |
| Human drawing | ❌ API/JSON only | ✅ Core use case |
| Real-time collaboration | ❌ | ✅ |
| Deterministic output | ✅ (same JSON = same sketch) | ❌ |
| Shareable sketch URL | ✅ | ✅ |
| Agent-readable output | ✅ (fetch URL → get JSON back) | ⚠️ |
| MCP server | ✅ First-party | ⚠️ Community only |
| Hand-drawn aesthetic | ✅ (multi-harmonic wobble) | ✅ |
| Architectural elements | ✅ (doors, windows, stairs, dimensions) | ⚠️ Basic shapes only |
| Free tier | ✅ | ✅ |
| Self-hostable | ✅ | ✅ |

## The Question That Decides It

One question determines which tool you need:

**"Is an AI agent generating this, or is a human drawing it?"**

If the answer is AI: Skissify.  
If the answer is human: Excalidraw.

If the answer is "both, at different stages": use both — they compose cleanly.
