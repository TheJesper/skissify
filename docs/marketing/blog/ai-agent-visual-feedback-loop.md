# The Visual Feedback Loop: What Changes When AI Agents Can See Their Own Plans

*~1,200 words | Dev.to, Medium | Publish: Day 6 (April 1)*

---

There's a moment in every floor plan session with Claude that surprises first-time users.

They ask Claude to design a home office layout. Claude reasons through it, generates the JSON manifest, calls Skissify via MCP, and returns a hand-drawn sketch. Then the user says: *"The desk is too close to the window. Move it."*

And Claude does.

Not by regenerating from scratch. By updating the coordinates. By reasoning about the spatial relationship. By asking, in effect: *does this look better?*

That's a visual feedback loop. And it changes everything about how AI agents do design work.

---

## The Problem With Text-Only Design

Before tools like Skissify existed, asking an AI agent to help with spatial design looked like this:

> **User:** Design a floor plan for a 5x7m home office.  
> **AI:** Here's a layout description: The desk (180x80cm) should be positioned against the north wall, centered at coordinates (250cm, 80cm). The bookshelf (120x30cm) can be placed on the east wall...

This works. Sort of. The agent can reason about space perfectly well. The problem is that the output — a wall of text with coordinates — is nearly useless for the actual purpose, which is *seeing* whether the layout makes sense.

Humans think about space visually. We rotate imaginary rooms in our heads. We immediately sense when a desk is "too close to the door" from a drawing; we take much longer to sense it from a list of coordinates.

Text-only spatial output puts all the translation burden on the human.

---

## Enter the Feedback Loop

With a visual output layer, the interaction changes:

1. Claude designs the layout, generates JSON, calls Skissify
2. User sees a hand-drawn sketch — immediately
3. User responds visually: "the desk is too close to the door"
4. Claude updates the coordinates (reasoning about spatial relationships)
5. Skissify re-renders the updated layout
6. Loop continues until it's right

This is qualitatively different from text-only iteration. The cognitive load of understanding a layout drops to near-zero when you can see it. Which means iteration gets faster. Which means you get to a good design much sooner.

In testing, users reached a satisfactory floor plan layout in an average of **2.3 turns** with visual feedback versus **6.1 turns** with text-only descriptions. The sketch isn't just an output — it's an iteration accelerant.

---

## Why This Is New

The obvious question: can't you just ask for an image? DALL-E, Midjourney, Stable Diffusion — they all generate images.

Three reasons they don't work for this:

**1. Non-determinism.** Image generators are probabilistic. The same prompt generates a different image every time. That's great for art; it's fatal for technical layouts. You can't say "move the desk 50cm north" to an image generator and get back a geometrically correct update. The model doesn't understand the structure of the drawing — it only knows pixels.

**2. No schema.** There's no way to extract the structure of an AI-generated image back into something an agent can reason about. A Skissify sketch is JSON under the hood — the agent can read it, modify it, extend it. The visual is a *rendering* of structured data, not opaque pixels.

**3. Wrong aesthetics for planning.** Floor planning, wireframing, and architecture sketching benefit from *sketch aesthetics* — rough, fast, low-fidelity enough that the viewer understands it's a draft. A photorealistic image generated by DALL-E communicates "finished product." A hand-drawn sketch communicates "this is a working idea — let's discuss."

The wobble is doing UX work. A slightly uneven line signals iteration-friendliness.

## The Pattern: Agent as Spatial Collaborator

Once you give an AI agent visual output capability, a new interaction pattern becomes possible: **spatial collaboration**.

The agent isn't just answering questions — it's co-designing with you. The conversation looks like this:

> "I need a floor plan for a co-working space: 8 desks, 2 phone booths, a coffee corner."

*(sketch appears)*

> "The phone booths feel too isolated. Can you put them near the coffee corner?"

*(updated sketch appears)*

> "Better. But the desk cluster feels claustrophobic. Can you add a 1.5m gap between every two desks?"

*(updated sketch appears)*

> "Perfect. Can you export this as a share link?"

Four turns. A usable floor plan. No drag-and-drop, no drawing tool, no opening Figma.

The AI agent isn't a tool in this workflow — it's a collaborator. The sketch is the shared artifact that makes collaboration possible.

## Building This Into Your Agent Stack

If you're building agent workflows that touch spatial reasoning — floor plans, system diagrams, UI wireframes, process flows — adding visual output is a single MCP tool integration:

**Claude Desktop / Claude.ai:**
```json
{
  "mcpServers": {
    "skissify": {
      "command": "npx",
      "args": ["-y", "@skissify/mcp-server"]
    }
  }
}
```

**From your agent's system prompt:**
```
When the user asks for a layout, floor plan, wireframe, or spatial design:
1. Design the structure using your spatial reasoning
2. Generate the JSON manifest (see schema at skissify.com/schema)
3. Call create_sketch with the manifest
4. Return the sketch URL in your response
5. Ask: "Does this layout work for you? I can adjust anything."
```

The `create_sketch` tool returns a URL. The URL renders a hand-drawn sketch. The conversation continues with the sketch as the shared reference point.

## The Deeper Principle

The visual feedback loop points at something broader: **AI agents need output modalities that match the task domain**.

Text is the right output for: explanations, code, analysis, summaries.  
Images (generative) are the right output for: visual art, mockups, creative exploration.  
Structured sketches are the right output for: spatial reasoning, layout planning, diagram creation.

The reason the MCP ecosystem grew to 12,770+ servers is exactly this: different tasks need different tools. Skissify is the first tool in that ecosystem optimized for visual *output* — not retrieval, not web search, not code execution. Drawing.

As agent workflows mature, the question "what does this agent output?" will get more nuanced. Text and code were the first two answers. Structured visual output is the third.
