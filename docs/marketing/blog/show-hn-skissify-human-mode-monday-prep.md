## EXPECTED OBJECTIONS AND RESPONSES

**"Why not just use Excalidraw?"**
> Excalidraw has no official MCP server as of March 2026. GlyphMCP (community wrapper) gets ~60-70% LLM first-try success. Skissify's schema was designed specifically for LLM generation and gets 94%. The difference is the schema constraints — flat structure, absolute coords, top-level aesthetics.

**"This is just calling Claude to call Claude"**
> Human Mode is a convenience layer. The JSON schema is still the core format — it's designed to be generated by any LLM, inspected, versioned, and shared. The text → JSON conversion is one step; the sketch rendering is the product.

**"Why not DALL-E/Midjourney?"**
> Image generators can't produce deterministic, iteratively-editable spatial output. A sketch URL from Skissify is permanent, inspectable, shareable — it's agent memory, not a generated image. The AI can revisit and modify it.

**"Is this open source?"**
> The rendering engine is open. The MCP server is public on npm. The hosted API is the commercial product.

**"JSON is verbose, why not a DSL?"**
> We tried. A DSL improves human authoring at the cost of LLM generation reliability. LLMs generate JSON at 94% success; our initial DSL attempts dropped that to ~65%. JSON is the right format for machine-generated input.

**"What's the pricing?"**
> Free tier: 50 sketches/month. Pro (EUR 5/mo): unlimited sketches, private storage, API access. We're considering raising prices — the current rate is very low for a developer API.
